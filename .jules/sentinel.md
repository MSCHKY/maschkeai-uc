## 2025-03-07 - Cross-Site Scripting (XSS) via HTML Element attributes
**Vulnerability:** The AI response rendering pipeline contained an XSS vulnerability due to improper HTML escaping. The `escapeHtml` function used the pattern `element.textContent = text; return element.innerHTML;` which effectively escapes `<` and `>` but **does not escape quotes (`"` and `'`)**. This allowed quotes to remain in the string. These unescaped strings were later injected directly into HTML attributes using a regex replacement (e.g., `data-cmd="$1"`).
**Learning:** Assigning a string to `.textContent` and then reading `.innerHTML` is not sufficient for escaping text that will be used inside HTML attributes. While it prevents tag injection, it allows attribute breakout if the text is later placed inside quotes.
**Prevention:** Always use a robust, regex-based escaping map that covers `&`, `<`, `>`, `"`, and `'` when escaping strings for general HTML use, especially if the string might be interpolated into HTML attributes.
